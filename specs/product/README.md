## PRD Template Structure

Each product spec should include:

1. **Overview** - Brief description of the feature
2. **User Stories** - Who uses it and how
3. **Requirements** - Functional requirements
4. **Acceptance Criteria** - How to verify it works
5. **Edge Cases** - Boundary conditions to handle
6. **Out of Scope** - What this spec does NOT cover
