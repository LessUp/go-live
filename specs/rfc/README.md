## RFC Template Structure

Each RFC should include:

1. **Summary** - Brief overview of the proposal
2. **Motivation** - Why this change/decision is needed
3. **Detailed Design** - Technical specification
4. **Alternatives Considered** - Other options and why they were rejected
5. **Implementation Plan** - How it will be built
6. **Open Questions** - Unresolved issues (if any)

---

## RFC Numbering

RFCs are numbered sequentially: `NNNN-short-title.md`

- `0001` - First RFC
- `0002` - Second RFC
- etc.
